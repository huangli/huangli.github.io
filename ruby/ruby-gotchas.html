method 1, 2
method(1, 2)
method (1, 2) ## ERROR

def method; 42; end
num = 21
method/num
method / num
method/ num

=== is confusing

like this:

1 === 1            => true
Fixnum === 1       => true
1 === Fixnum       => false

Fixnum === Fixnum  => false

(1..3) === 2       => true
2 === (1..3)       => false

Remember, === used to test equality within a when clause of a case statement.

(1..4) === 5       => false

Operator precedence

x = true & false
x => false

x = true and false
x => true

Regexps is not same

In "standard" regexps:

`^` is start and `$` is end of the whole string.

Ruby's regexps default to multiple line, so:

^ and $ is start and end of any line,
A and E is start and end of whole string.

getting.any?

[].any?            => false
[1].any?           => true
[:foo, :bar].any   => true

# ok so far, BUT:
[false].any?       => false
[nil].any?         => false
[false, nil].any?  => false
